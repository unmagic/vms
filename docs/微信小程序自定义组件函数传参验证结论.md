# 微信小程序自定义组件函数传参验证结论

> 更新时间：2026-04-23
> 验证环境：模拟器 + 真机（iOS / Android），结果一致

---

## 结论总览

| 场景                                                        | 结果             | 说明                                     |
| ----------------------------------------------------------- | ---------------- | ---------------------------------------- |
| `onLoad` 内定义的函数，通过 `setData` 传入 property         | ✅ **能传递**    | 运行时动态创建并赋值                     |
| `type: Object` 包裹 `{ fn: Function }`，通过 `setData` 传入 | ✅ **能传递**    | 同上，对象内的函数引用同样有效           |
| Page 对象同级定义的方法，通过 `setData` 传入                | ❌ **始终 null** | 框架层面的方法引用无法通过 property 传递 |

---

## 详细说明

### ✅ 场景 1：onLoad 内定义函数，通过 setData 传入

```js
// 父页面
Page({
  data: {
    onCallback: null,
  },

  onLoad() {
    const fn = (arg) => {
      console.log('收到:', arg)
      return '返回值'
    }
    this.setData({ onCallback: fn })
  },
})
```

```xml
<!-- 父页面 wxml -->
<fn-receiver on-callback="{{onCallback}}" />
```

```js
// 子组件 component
Component({
  properties: {
    // function参数的类型只能使用null来定义
    onCallback: { type: null },
  },

  attached() {
    console.log(typeof this.properties.onCallback) // "function" ✅
  },
})
```

**结论：能正常传递，组件内可调用。**

---

### ✅ 场景 2：Object 包裹函数，通过 setData 传入

```js
// 父页面
Page({
  data: { handler: null },

  onLoad() {
    this.setData({
      handler: {
        fn: (arg) => {
          console.log('handler fn 被调用:', arg)
          return 'done'
        },
        label: 'test',
      },
    })
  },
})
```

```xml
<fn-receiver handler="{{handler}}" />
```

```js
// 子组件
Component({
  properties: {
    handler: { type: Object, value: null },
  },

  attached() {
    console.log(typeof this.properties.handler?.fn) // "function" ✅
  },
})
```

**结论：对象内的函数引用同样有效，`handler.fn` 可正常调用。**

---

### ❌ 场景 3：Page 同级方法，通过 setData 传入

```js
// 父页面
Page({
  // 和 onLoad 同级的方法
  myCallback(arg) {
    console.log('myCallback 被调用:', arg)
  },

  onLoad() {
    this.setData({ onCallback: this.myCallback })
  },
})
```

```xml
<fn-receiver on-callback="{{onCallback}}" />
```

```js
// 子组件
Component({
  properties: {
    onCallback: { type: null },
  },

  attached() {
    console.log(this.properties.onCallback) // null ❌
    console.log(typeof this.properties.onCallback) // "object"（实际为 null）
  },
})
```

**结论：传入后始终为 null，无法调用。**

---

## 原因分析

微信小程序对 `properties` 的函数类型，在**逻辑层内部**做了特殊处理——函数引用被保留，并不走完整的 JSON 序列化丢弃。这解释了为什么场景 1、2 能正常工作。

但 **Page 同级方法**（如 `myCallback`）是框架层面的方法引用，属于 Page 实例的方法，不属于数据流的一部分。`setData` 时这类引用被过滤，始终为 null。

这说明：

- **函数能否传递，取决于它是否处于"数据"上下文中**，而非函数本身是否可序列化
- 动态创建的闭包函数可以被传递，Page 方法引用不能被传递
- 模拟器和真机行为一致，是框架的稳定行为

---

## 对 VMS 的影响

VMS 编译 `:request="fn"` 为 `{ fun: fn }` 的方案：

- **可行**：如果函数是在组件 `setup` / `onLoad` 等生命周期内动态定义并 `setData` 传入，即走数据流路径，可以正常工作
- **需注意**：需要确认编译产物走的是 `setData({ request: fn })` 的数据路径，而非直接绑定到 WXML 属性字面量的方式
